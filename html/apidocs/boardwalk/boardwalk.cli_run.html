boardwalk.cli_run#

run and check CLI subcommands

Module Contents#

Functions#

run

Runs workflow jobs defined in the Boardwalkfile.py

check

Runs workflow in check mode. Equivalent to run –check

run_workflow

Runs the workspace’s workflow against a list of hosts

run_failure_mode_handler

Common failure handler that used in catching exceptions during workflow runs

filter_hosts_by_limit

Accepts a list of host objects and returns a list of object matching a host pattern string

sort_host_list

Accepts a list of host objects and return a list of host objects sorted by the requested sort method. The acceptable sorting methods are/must be defined and validated by the calling click function

check_host_preconditions_locally

Checks preconditions of jobs defined in workspace’s workflow. Returns a list of the hosts that meet preconditions. Prints warnings for hosts that don’t pass. Preconditions are ignored for hosts that have started a workflow but haven’t finished

handle_workflow_catch

Handles local and remote workspace catches. Blocks under caught conditions

lock_remote_host

bootstrap_with_server

Performs all of the initial set-up actions needed when boardwalk is configured to connect to a central boardwalkd

update_host_facts_in_local_state

Updates fetches latest host facts for a host and saves to the workspace state

directly_confirm_host_preconditions

Connects directly to the host and confirms that the workflow job preconditions are actually met. Raises an exception if any are unmet. If a workflow was run but never completed, preconditions are ignored. Also posts events to the central server

execute_workflow_jobs

Executes workflow jobs. Different kinds of job types are specified

execute_host_workflow

Handles executing all jobs defined in a workflow against a host
