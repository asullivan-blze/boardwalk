Base request handler for API paths

AuthLoginApiHandler

Handles authenticating a user to the API and sending back a token to the a client with an open login socket. This handler uses the UIBaseHandler intentionally because the user must visit the UI to authenticate here

AuthLoginApiWebsocketHandler

Socket used by CLI clients to login to the API and get an auth token

AuthApiDenied

Dedicated handler for redirecting an unauthenticated user to an ‘access denied’ endpoint

WorkspaceCatchApiHandler

Handles setting a catch on a workspace

WorkspaceDetailsApiHandler

Handles getting and updating WorkspaceDetails for workspaces

WorkspaceHeartbeatApiHandler

Handles receiving heartbeats from workers

WorkspaceEventApiHandler

Handles events sent from clients to the server. Events are always logged to the server’s stdout and a limited number of events are visible in the UI. Optionally the client can request the server “broadcast” an event message, and the server will post the broadcasted message to slack as well, if a slack webhook is configured

WorkspaceMutexApiHandler

Handles workspace mutex api requests

WorkspaceSemaphoresApiHandler

Handles getting server-side WorkspaceSemaphores
