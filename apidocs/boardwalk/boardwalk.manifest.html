class boardwalk.manifest.WorkspaceConfig(host_pattern: str, workflow: boardwalk.manifest.Workflow, default_sort_order: str = 'shuffle', require_limit: bool = False)#

Configuration block for workspaces

Parameters:

  • default_sort_order – The default order hosts will be walked through (by hostname). Valid sort orders are specified in the valid_sort_orders attribute

  • host_pattern – The Ansible host pattern the workspace targets. If this changes after initialization, the workspace needs to be re-initialized

  • require_limit – check and run subcommands will require the –limit option to be passed. This is useful for workspaces configured with a broad host pattern but workflows should be intentionally down-scoped to a specific pattern

  • workflow – The workflow the workspace uses

Initialization

class boardwalk.manifest.Workspace#

Bases: abc.ABC

Handles everything to do with the active workspace directory. “Workspaces” are used to hold configuration and temporary information. This is done so different upgrade projects are possible to do in parallel. This class implements the singleton pattern, because the same Workspace may be instantiated in many areas
